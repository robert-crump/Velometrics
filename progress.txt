## Issue #6 — Move "POIs nearby" from "Navigate" to "Map"
Completed: 2026-06-04

### What was done

**Shared component extraction**
- Added `categoryDisplayName()` and `formatPoiDistance()` to `FormatUtils.kt` (promoted from
  private functions in `NavigationScreen`)
- Created `ui/components/PoiPopupCard.kt`: shared composable displaying POI name, category,
  opening hours (conditional), air distance (conditional on GPS), and "Open in Maps" button;
  `openPoiInGoogleMaps()` top-level function alongside it

**POI layer on Map tab (`MapViewScreen` / `MapViewViewModel`)**
- `MapViewViewModel`: lazy-loads all POIs from repository on first toggle of the POI layer;
  derives `visiblePois` by combining `_allPois`, `_viewportBounds`, and `_selectedPoiCategories`
  via `combine`; `selectPoiFromMap()` computes air distance from current GPS fix (null if
  unavailable); `updateViewportBounds()` called by the screen's camera idle listener
- `MapViewScreen`: registers `addOnCameraIdleListener` in `onMapReady` to keep viewport bounds
  in sync; `LaunchedEffect` for POI rendering and highlight; extended map click listener handles
  cluster zoom-in, individual POI popup, then falls through to existing interval tap logic;
  `PoiPopupCard` shown at bottom center when a POI is selected
- Layers bottom sheet: POI toggle row; indented `Checkbox` sub-rows per category, only visible
  when the POI layer is on; all categories activated by default when layer first enabled;
  relies on MapLibre clustering (radius 50, maxZoom 14) instead of a zoom floor

**Remove NEARBY mode from Navigate**
- `NavigationViewModel`: removed `PoiMode` enum, `selectedMode`, `fetchPoisNearbyByRadius()`,
  `nearbyRadiusM`/`setNearbyRadius()`; Navigate is now always in ALONG_TRACK mode;
  `setUserPositionFromPermission()` and `refreshDistances()` simplified accordingly
- `NavigationScreen`: removed `ModeSelectionScreen` composable and mode-selection flow;
  screen opens directly to LIST/MAP tabs; GPX file picker moved to a Route icon in the
  TopAppBar actions; permission request triggered on GPX load via `LaunchedEffect(gpxTrack)`
- `NavigationViewModelTest`: removed 4 NEARBY-specific tests; updated the ALONG_TRACK
  permission-fallback test (no longer requires `setMode()`)

## Issue #14 — Navigate: Bugfixes
Completed: 2026-06-05 | Closed: 2026-06-05

### What was done

**Remove "cuisine" from POI row**
- Removed `poi.cuisine?.let { append(" ($it)") }` from `PoiRow` in `NavigationScreen`; category
  label is now shown without the cuisine qualifier

**Bottom drawer handle always visible**
- Replaced the fixed `rememberStandardBottomSheetState(skipHiddenState = false)` with a `key(gpxLoaded)`
  block that recreates the state when GPX load state changes
- Pre-GPX: `skipHiddenState = false`, `initialValue = Hidden` (no handle shown)
- Post-GPX: `skipHiddenState = true`, `initialValue = PartiallyExpanded` (handle always visible,
  user cannot drag sheet fully hidden)
- `LaunchedEffect(gpxTrack)` simplified to only launch the location permission request; sheet
  expansion is now handled by `initialValue`

**Full-screen mode overlap fix**
- Added `sheetExpansionPadding` (= `outerPadding.calculateTopPadding()` when expanded, 0 otherwise)
  applied to both `sheetDragHandle` and `sheetContent` Column, keeping handle and buttons clear
  of the top app bar when the sheet is fully expanded

**No look-ahead active by default; look-ahead buttons control camera + list**
- `_lookAheadOption` changed from `StateFlow<LookAheadOption>` (defaulting KM5) to
  `StateFlow<LookAheadOption?>` (defaulting null)
- Added `allTrackPois: List<PoiWithDistances>` internal cache; full track POIs are fetched once
  on GPS fix and stored here
- `setLookAheadOption(option)` now toggles: clicking the active button deselects (→ null);
  POI list is filtered in-memory from `allTrackPois` (no repo re-fetch); `pendingCameraBounds`
  is set to the appropriate `LatLngBounds` (look-ahead sub-track bounds or full-track bounds)
- `LaunchedEffect(pendingCameraBounds, mapAndStyle)` in screen animates via `easeCamera` at 600ms
  then calls `consumeCameraFit()`
- Behind-user POIs get a signed negative `trackDistanceM`; the distance label in `PoiRow` is
  suppressed for negative values (`takeIf { it >= 0.0 }`)

**Off-track dialog**
- `checkOffTrack()` fires once per GPX load (guarded by `hasCheckedOffTrack` flag) after the
  first GPS fix; uses `TrackGeometryUtils.projectPointOntoTrack` to measure perpendicular distance
- If distance > 1000 m, `_offTrackDialogKm` is set → `AlertDialog` shown with exact km distance
  and a single Dismiss button; `dismissOffTrackDialog()` clears the state

### Tests
100 unit tests pass.
